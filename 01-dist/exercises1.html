Exercise set 1

Practice exercise

The first exercise given here, Max, is included for practice and to help you get accustomed to what will be provided to you, for each exercise; it does not need to be submitted. Let’s first take a look at the exercise description:

  • Max (Max.java).

    Write a method max that, given two ints, returns the maximum of the two given values.

Note that, after the name of the exercise (in this case, Max), the name of the file where you should provide your code is given (in this case, this name is Max.java). You should be able to find a file called Max.java included with the files for this exercise set. Unless explicitly mentioned, the method header(s) should be set up for you. You should not modify the method header(s) nor the class header(s). You should provide code in the method body or bodies. Typically, a “checker” will be provided, and the name of the file where it is provided will be the filename given in the description with “Checker” added before “.java”; in this case, the checker is provided in the file MaxChecker.java. (To work with these files in IntelliJ, they should be placed into a project; depending on your version, it might be possible to create an empty project, and then, in your operating system, copy, into the src directory of the project, the .java files that you wish to include in the project.)

Please try running the main method of MaxChecker.java without modifying the code of the method max, so that the method body of max simply contains “return 0;”. You should see something like the following, as output:

---- On code snippet:
{ s = "" + Max.max(3,4); }
---- Expected value of String s:
4
---- Actual value of String s:
0

The “code snippet” shows a snippet of code where the given code has been judged to behave incorrectly (here, s is declared to be a String prior to the snippet). The expected (correct) value of s is provided (here, it is 4), and the actual value of s that resulted from using your code (in Max.java) is also provided (here, it is 0). Now, try modifying the body of the method max so that it contains “return 4;”. You should then see a different code snippet where this second implementation fails, along with the other described information. Please also try placing code that correctly returns the maximum of the given ints in the body of the method max, for example, the following code:

    public static int max( int a, int b )
    {
        if( a > b )
        {
            return a;
        }
        else
        {
            return b;
        }
    }

When you run the checker with this correct code, it should provide you with a message stating that you passed all of its tests. (Remark: you don’t actually have to read the code of the checker; you can just run its main method to use it.)

Note that the checker is intended to provide some basic tests, and there is certainly no guarantee that your code is correct if it passes all of the checker’s tests. You are strongly encouraged to test your own code. One way to do this is to write your own testing code in a main method of the file where you are supposed to provide your code (in the case of Max.java, you can see that a main method with empty body has been provided.) For these exercises, any testing code that you write (that is not invoked by the methods you are asked to implement) will not be considered when performing mark evaluation.

Exercises to be submitted

The descriptions of the exercises that do need to be submitted are given in the following. Please submit one .zip file that contains just the solutions to these exercises (with the specified filename); the first two lines of each .java file you submit must contain end-of-line comments containing your 8-digit Birkbeck ID, and your e-mail address. The first line should consist of two slashes (/), a space, and your 8-digit Birkbeck ID; the second line should consist of two slashes (/), a space, and your e-mail address. Please note that (1) the method that you are asked to implement should not print anything to the screen (unless mentioned explicitly), and (2) you are asked not to use packages in the .java files that you submit, i.e., each .java file should not contain the keyword “package” at all.

  • Middle of three (Middle.java).

    Write a method middle that, given three ints, a, b, and c, returns the middle value: if two (or more) of the given ints are equal to a value x, the returned value should be that value x; otherwise, the returned value should be the unique value that is strictly larger than one of the other values and strictly smaller than one of the other values.

  • Intersecting intervals (Intervals.java).

    When a and b are integers, the notation [a,b] is used to denote the set of all integers x such that a <= x and x <= b (that is, such that a is less than or equal to x, and x is less than or equal to b). A set of this form is called an interval.

    The intersection of any two given intervals (that is, the set of integers that fall into both of the intervals) is always an interval. Write a method intersect that, given four ints a1, b1, a2, and b2, returns a String, as follows:

    • If the intersection of the intervals [a1,b1] and [a2,b2] is non-empty (that is, if this intersection contains one or more integers), the returned String should hold a value presenting this intersection using the introduced notation.
    • If this intersection is empty, the returned String should be “[]”.

    For example, upon being given (1,3,2,4), the method should return the String “[2,3]”: this is because [2,3] is the intersection of [1,3] and [2,4].

  • Add one second (AddSecond.java).

    Write a method addSecond that, given three ints h, m, and s representing the hours, minutes, and seconds of a clock time in 24-hour format, returns a String containing the given clock time plus one second. You can assume that the given ints obey 0 <= h < 24, 0 <= m < 60, and 0 <= s < 60. Your method should always return a clock time in hours:minutes:seconds format where the hours, minutes, and seconds obey the constraints given in the previous sentence, and where each of these three components is represented using exactly two digits.

    For example, upon being given (19,45,59), the method should return the String “19:46:00”.
